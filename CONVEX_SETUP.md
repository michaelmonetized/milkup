# Convex Setup Instructions

## For Production Deployment

The WYSIWYG editor code is complete and ready to use with Convex Cloud. Follow these steps to finish the setup:

### 1. Interactive Convex Initialization

```bash
cd ~/Projects/milkup

# This will prompt you interactively
bunx convex init
```

**When prompted:**
- **Team:** Choose `hustle-testing`
- **Project:** Choose `milkup-editor` (or create new)
- **Auth info:** Press Enter to use `node` (already set)

This generates the Convex deployment URL and updates `.env.local` automatically.

### 2. Deploy Schema to Convex Cloud

```bash
bunx convex deploy
```

This pushes the media schema to your Convex project.

### 3. Get Your Deployment URL

After `convex init`, check your `.env.local`:

```bash
cat .env.local
```

You should see:
```
NEXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
```

Copy this URL — it's your production endpoint.

### 4. Seed Sample Media (Optional)

In Convex dashboard, add sample media items:

```javascript
// Register in Convex database
{
  filename: "sample.jpg",
  type: "image",
  url: "https://images.unsplash.com/...",
  source: "convex",
  uploadedAt: Date.now()
}
```

This populates the media picker modal.

### 5. Test Locally

```bash
# Dev server with Convex backend
bunx next dev

# Open http://localhost:3000
# Test features:
# - Click toolbar buttons → markdown updates
# - Paste from Google Docs → auto-converts
# - Click "Insert Media" → fetches from Convex
# - Save Draft → localStorage
```

### 6. Deploy to Production

When ready:

```bash
# Push to GitHub (auto-deploys to Vercel)
git push origin master

# Vercel will:
# 1. Build project
# 2. Use Convex production deployment
# 3. Deploy to https://your-domain.vercel.app
```

## Troubleshooting

**"Cannot find Convex project"**
- Ensure `hustle-testing` team is active in your Convex account
- Run `bunx convex projects` to list available projects
- Create new project if needed: `bunx convex project create milkup-editor`

**"NEXT_PUBLIC_CONVEX_URL not set"**
- Run `bunx convex init` again (generates .env.local)
- Verify `.env.local` exists with URL
- Restart dev server after updating .env.local

**"Media picker shows no items"**
- Seed sample data in Convex dashboard
- Or upload through the app (file upload feature — future)
- Check Convex logs for query errors
